Dear Editors,

We submit for consideration the manuscript "cnefetools: Access and
Analysis of Brazilian CNEFE Address Data in R", describing the
cnefetools package, which is currently available on CRAN (v0.2.2).

THE PROBLEM WE ADDRESS

The 2022 Brazilian CNEFE (National Address File for Statistical
Purposes) is a geocoded register of approximately 100 million address
points covering all of Brazil's ~5,570 municipalities. Despite its
enormous analytical potential for urban research and public policy
analysis, the dataset has remained inaccessible to most researchers due
to the following barriers:

  - Data are distributed as individual ZIP-compressed CSV files (one
    per municipality) via an IBGE FTP server, with no unified API.
  - Files are semicolon-delimited and Latin-1-encoded; some columns are
    inconsistently typed across municipalities.
  - Spatial operations on millions of geocoordinates (H3 cell
    assignment, point-in-polygon joins) are computationally expensive
    without specialised tooling.
  - No existing R package provided programmatic access to this dataset.

OUR CONTRIBUTION

cnefetools provides a unified, documented R interface for the CNEFE
with four main capabilities:

  1. Data access: download and cache CNEFE data for any municipality
     using the standard R user cache directory, with transparent
     management of the IBGE FTP structure.

  2. Address aggregation: count addresses by type within H3 hexagonal
     grids or user-supplied polygons, using a dual DuckDB/R backend
     that achieves speedups of up to 20x over pure R for large cities.

  3. Land-use mix indices: compute six established and novel land-use
     mix indices (including the Bidirectional Global-centered Balance
     Index, BGBI) directly from geocoded address data.

  4. Dasymetric interpolation: redistribute census tract-level
     statistics to finer spatial units using CNEFE dwelling points as
     ancillary data.

A key architectural contribution is the dual-backend design: the
default DuckDB engine reads CSV data directly from cached ZIP archives
via the zipfs extension, performs H3 cell assignment and spatial joins
entirely in-process (via the h3 and spatial extensions), and returns
only the aggregated result to R. This design reduces processing time
for São Paulo (~5.7 million addresses) from several minutes to seconds.

WHY THIS IS RELEVANT TO THE R JOURNAL

  - The package fills an important gap for a major country's address
    data: Brazil is the 5th most populous country and the CNEFE is the
    first R-accessible, reproduction-ready interface to its geocoded
    address register.
  - The dual-backend DuckDB/R architecture illustrates a design pattern
    of broad interest to the R community working with large geospatial
    datasets.
  - The dasymetric interpolation function (`tracts_to_h3()` /
    `tracts_to_polygon()`) implements an in-process spatial workflow
    that may inform similar packages for other national census systems.
  - The package includes six pre-rendered pkgdown articles, full test
    coverage, and an active issue tracker.

RELATIONSHIP TO COMPANION PUBLICATION

A companion manuscript (Pedreira Jr. et al., 2025, engrXiv doi:
10.31224/5975) presents the methodological derivation of the Bidirectional
Global-centered Balance Index (BGBI) and evaluates it against existing
land-use mix measures using the CNEFE data. That paper focuses on the
statistical methodology; this R Journal submission focuses on the
software. The two papers cite each other and are complementary, with
no content duplication.

REPRODUCIBILITY

The worked example in Section 6 uses Vitória da Conquista (IBGE code
2933307, ~200,000 addresses) and completes in under ten minutes from
scratch on a modern laptop, satisfying the R Journal reproducibility
requirement. All code chunks are self-contained and require only CRAN
packages.

We believe cnefetools meets the R Journal's criteria for contributions
that make significant statistical methodology or data-analysis
capabilities accessible to the R community. We look forward to the
editors' and reviewers' feedback.
